## 编号

BUG-004

## 类型

- [x] BUG

## 模块

API / History

## 描述

`HistoryServiceImpl.buildVO` 中硬编码封面 URL 为 `/comic/thumbs/{id}/cover.webp`，但项目 URL 规范为 `/files/thumbs/{relativePath}`（由 `FileUrlResolver` 统一生成），导致 Reading Center 封面 404。

## 复现步骤

1. 确保存在阅读历史
2. 打开 `/history`
3. 浏览器 Network 观察到封面请求 `/comic/thumbs/{id}/cover.webp` 返回 404

## 期望行为

`HistoryVO.coverUrl` 应使用与 Library/Detail 一致的 URL 规范。

## 实际行为

封面 URL 与 Nginx 路由不匹配，无法加载。

## 影响范围

- 是否阻塞 Beta v0.1 发布？是
- 是否影响全链路？是，History 页面视觉完整性受损

## 建议修复方案

方案 A（推荐最小改动）：后端 `HistoryServiceImpl.buildVO` 改为 `/files/thumbs/${comic.id}/cover.webp`（假设 thumbs 根下按 comicId 组织）。

方案 B：统一使用 `FileUrlResolver` 生成封面 URL（若 thumbs 也纳入 StorageRoot 体系）。

先按方案 A 处理，确保与 nginx `/files/thumbs/` alias 匹配；后续若 thumbs 路径规则变化再统一改。
