## 编号

BUG-007

## 类型

- [x] BUG

## 模块

Import / Docker

## 描述

前端用户输入的是宿主机路径（例如 `D:/manga/hq/38`），但 Worker 运行在 Docker 容器内，容器通过 volume 把宿主机 `D:/manga` 挂载到 `/storage`。Worker 直接拿前端传入的 `sourcePath` 去访问，在容器内找不到 `D:/manga/hq/38`，导致导入失败。

## 复现步骤

1. Docker Compose 启动 Worker
2. 从前端创建 DIRECTORY 导入，路径填 `D:/manga/hq/38`
3. Worker 收到任务后访问该路径失败

## 期望行为

前端输入宿主机路径，Worker 能正确映射到容器内部路径。

## 实际行为

Worker 在容器内访问宿主机绝对路径，文件不存在。

## 影响范围

- 是否阻塞 Beta v0.1 发布？是
- 是否影响全链路？是，Docker 部署下导入不可用

## 建议修复方案

方案 A：Worker 配置 `manga.root.host` 与 `manga.root.container`，在 `ImportTaskHandler` 中将 sourcePath 中的 host root 替换为 container root。

方案 B：前端导入表单提示用户输入容器内路径（体验差，不推荐）。

方案 C：Worker 不依赖宿主机路径，改用网络存储或统一约定 `/storage` 前缀。

推荐方案 A，保持前端用户体验不变。

## 修复记录

- 在 `WorkerConfig` 新增 `hostMangaRoot` 与 `containerMangaRoot`（默认 `/storage`）。
- 在 `ImportTaskHandler` 中增加 `mapHostPathToContainer()`，当 `sourcePath` 以 `hostMangaRoot` 开头时替换为 `containerMangaRoot`。
- 在 `docker-compose.yml` 中为 `worker-service` 设置 `WORKER_HOST_MANGA_ROOT=${MANGA_ROOT:-./manga}` 与 `WORKER_CONTAINER_MANGA_ROOT=/storage`。
- 同步修复 `api-service` 未挂载 `metadata`/`hq` 卷导致无法读取 Worker 写入的 `metadata.json` 问题。
- 验证：传入 `D:/manga/hq/xxx` 宿主机路径，Worker 正确映射为 `/storage/hq/xxx` 并完成导入。
