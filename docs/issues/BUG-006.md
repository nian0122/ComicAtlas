## 编号

BUG-006

## 类型

- [x] BUG

## 模块

Worker / RabbitMQ

## 描述

Worker 消费 `ImportTaskCreatedEvent` 时反序列化失败：

```
InvalidDefinitionException: Cannot construct instance of `java.time.Instant`
```

原因是 `ComicEvent` 接口包含 `Instant occurredAt()` 字段，而 Worker 的 `Jackson2JsonMessageConverter` 使用的 `ObjectMapper` 没有注册 `JavaTimeModule`，无法反序列化 Java 8 日期时间类型。

## 复现步骤

1. 启动完整 Docker Compose
2. 创建任意导入任务
3. Worker 日志出现 `InvalidDefinitionException` 并不断重试
4. 任务状态永远停留在 PENDING

## 期望行为

Worker 能正确反序列化事件并执行导入。

## 实际行为

反序列化失败，消息被不断重投，任务不推进。

## 影响范围

- 是否阻塞 Beta v0.1 发布？是
- 是否影响全链路？是，Import → Task 链路完全断裂

## 建议修复方案

在 `worker-service/src/main/java/com/comicatlas/worker/config/RabbitMqConfig.java` 的 `messageConverter()` 中显式配置 ObjectMapper 并注册 `JavaTimeModule`：

```java
@Bean
public MessageConverter messageConverter() {
    ObjectMapper mapper = new ObjectMapper();
    mapper.registerModule(new JavaTimeModule());
    return new Jackson2JsonMessageConverter(mapper);
}
```

同时检查 api-service 的 MessageConverter 是否也需要统一配置。
