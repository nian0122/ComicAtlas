## 编号

BUG-003

## 类型

- [x] BUG

## 模块

Task Center

## 描述

`TaskCard.vue` 的取消按钮在 `PARSING`、`PENDING`、`DOWNLOADING` 状态都会显示，但后端 `ImportServiceImpl.cancelTask` 只允许 `PENDING` 和 `DOWNLOADING` 状态取消。用户点击 PARSING 任务的"取消"会收到 400 错误。

## 复现步骤

1. 创建一个导入任务
2. 当任务状态变为 `PARSING`（Worker 已开始处理但尚未进入下载）
3. 点击取消
4. 后端返回 400

## 期望行为

前端只在真正可取消的状态显示取消按钮，或对不可取消状态禁用/隐藏。

## 实际行为

PARSING 状态显示取消按钮但后端拒绝。

## 影响范围

- 是否阻塞 Beta v0.1 发布？否
- 是否影响全链路？否，但用户体验差

## 建议修复方案

修改 `TaskCard.vue` 的 `canCancel` 为：

```ts
const canCancel = computed(() =>
  props.task.status === 'PENDING' || props.task.status === 'DOWNLOADING'
)
```
