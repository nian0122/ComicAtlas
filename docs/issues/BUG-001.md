## 编号

BUG-001

## 类型

- [x] BUG

## 模块

Reader

## 描述

从 Comic Detail 的"继续阅读"或 History 的"继续阅读"跳转到 Reader 时，URL 中携带了 `page` 查询参数（例如 `?chapterId=5&page=18`），但 ReaderPage 没有读取该参数，导致总是从 history 恢复页码或回到第 1 页。

## 复现步骤

1. 进入任意漫画的 Comic Detail
2. 确保该漫画有阅读历史（例如第 18 页）
3. 点击"继续阅读"
4. URL 形如 `/comics/1/read?chapterId=5&page=18`
5. Reader 加载后并未定位到第 18 页

## 期望行为

Reader 应优先使用 URL 中的 `page` 参数；没有时才调用 `restoreProgress()` 从历史记录恢复。

## 实际行为

总是调用 `restoreProgress()`，URL 中的 `page` 被完全忽略。

## 影响范围

- 是否阻塞 Beta v0.1 发布？是
- 是否影响全链路？是，Detail / History → Reader 的页码恢复链断裂

## 建议修复方案

在 `ReaderPage.vue` 的 `onMounted` 中：

1. 读取 `route.query.page`
2. `loadChapter` 后先尝试设置 `store.currentPage = pageFromQuery`
3. 若 query 中无 page，再调用 `store.restoreProgress()`
4. 调整 `reader-store.loadChapter` 不要每次都重置 `currentPage = 1`
