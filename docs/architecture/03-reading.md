# ComicAtlas 0.2 阅读模块设计

**版本**: 0.2  
**日期**: 2026-07-16  
**状态**: Canonical

---

## 设计目标

阅读模块保持极简、快速，不掺杂任何管理能力。用户进入阅读路径后，只看到与阅读相关的内容。

---

## 阅读流程

```
阅读首页 / 漫画库 / 历史
        │
        ▼
    漫画详情
        │
        ▼
    阅读器
        │
        ▼
    历史自动保存
```

只有这一条主流程。

---

## 页面职责

### ReadingHome (`/`)

阅读的默认入口，按优先级展示：

1. **继续阅读**：当前有阅读进度时，展示最近一本漫画和继续按钮。
2. **最近阅读**：按时间倒序展示最近阅读过的漫画。
3. **最近加入**：展示最近导入的漫画。
4. **漫画库**：完整漫画墙的第一页，支持无限滚动。

原则：首页不是 Library 的预览，而是阅读场景的自然起点。

### ComicLibrary (`/library`)

完整的漫画浏览页面：

- 漫画墙（网格/列表）。
- 搜索：标题、作者、标签。
- 筛选：Category、Tag、状态。
- 排序：最近阅读、最近加入、标题、作者。

### ComicDetail (`/comic/:id`)

只展示消费信息：

- 封面
- 标题、副标题、作者
- Category、Tag
- 简介（可折叠）
- 继续阅读 / 开始阅读
- 目录
- 历史

**不包含**：编辑、删除、修改封面、修改标签、修改分类。

### ReaderPage (`/reader/:chapterId`)

沉浸式阅读器：

- 按 `global_order` 线性翻页。
- 支持 HQ / LQ 切换。
- 自动保存阅读历史。
- 章节间 prev/next 导航。

### HistoryPage (`/history`)

阅读历史：

- 按漫画聚合，展示阅读进度。
- 支持删除单条历史。
- 支持继续阅读。

---

## UX 原则

1. **无管理按钮**：阅读页面不出现任何编辑、删除、导入、设置入口。
2. **继续阅读优先**：首页第一屏优先帮助用户回到上次阅读位置。
3. **最少的点击**：从打开应用到继续阅读， ideally 一次点击。
4. **一致的 Reading Layout**：顶部导航在所有阅读页面保持一致。
