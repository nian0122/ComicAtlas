# ComicAtlas v0.1.0 Release Notes

**发布日期**：2026-07-08

**定位**：MVP（Minimum Viable Product）—— 个人本地漫画仓库的端到端闭环。

---

## 核心功能

### 导入（Import）

- 支持 **ZIP** 与 **本地目录（DIRECTORY）** 两种来源。
- 创建导入任务后通过 RabbitMQ 异步处理。
- Worker 解析目录结构、搬文件到统一 MANAGED 存储、写入 `metadata.json`。
- API 消费完成事件，写入 `comic` / `catalog` / `chapter` / `page`。

### 任务中心（Task Center）

- 实时查看导入任务状态：PENDING / PARSING / IMPORTING / SUCCESS / FAILED。
- 进行中任务显示进度条；失败任务显示错误信息 + 重试按钮。
- TopNav 徽章显示进行中任务数量，全局轮询自适应频率。

### 漫画库（Library）

- Netflix 风格暗色卡片列表，展示封面、标题、作者、页数。
- 状态筛选、排序、搜索。
- 封面加载失败时显示占位图。

### 漫画详情（Comic Detail）

- Hero 区展示封面、标题、元信息、标签。
- 有阅读历史时显示「继续阅读」+ 进度条；无历史时显示「开始阅读」。
- Catalog Tree 展示章节结构，点击章节进入 Reader。
- 操作菜单：生成 LQ / 删除漫画。

### 阅读器（Reader）

- 进入指定章节与页码。
- 上一章 / 下一章导航。
- 键盘快捷键（方向键、空格）翻页。
- 离开页面时保存阅读进度（chapterId + pageNumber）。
- 刷新后可通过 History「继续阅读」恢复。

### 阅读中心（History / Reading Center）

- 按「今天 / 昨天 / 更早」分组。
- 显示封面、标题、章节、页码、百分比、相对时间。
- 点击「继续阅读」回到 Reader 对应页码。

## 基础设施

- **后端**：Spring Boot 3 + Spring Cloud Gateway + Nacos 服务发现。
- **消息队列**：RabbitMQ，统一事件规范 + DLX/DLQ 死信。
- **存储**：统一 `D:/manga/hq/{comicId}/{chapterId}/` MANAGED 布局；Nginx 提供 `/files/{root}/{path}` 静态访问。
- **数据库**：MySQL 8 + MyBatis Plus。
- **缓存**：Redis。
- **部署**：Docker Compose 一键启动（mysql / redis / rabbitmq / nacos / gateway / api-service / worker-service / nginx）。
- **测试**：Playwright E2E（smoke / import / reader / history）4/4 通过。

## 已知限制（Beta 期间不修复）

- LQ 需手动触发，不会自动生成。
- Reader 暂不支持预加载、双页、缩放、全屏、阅读方向切换。
- Dashboard 数据图表未实现。
- 设置页面未实现。
- 历史记录删除未实现。
- 批量取消 / 批量重试未实现。

## 快速开始

```bash
# 1. 准备 .env（示例）
echo "MANGA_ROOT=D:/manga" > .env

# 2. 启动全部服务
docker-compose up -d

# 3. 导入一本漫画
curl -X POST http://localhost:8000/api/tasks/import \
  -H "Content-Type: application/json" \
  -d '{"sourceType":"DIRECTORY","sourcePath":"D:/manga/hq/your-comic"}'

# 4. 打开前端
open http://localhost
```
