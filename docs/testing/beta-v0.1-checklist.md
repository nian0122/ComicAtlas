# ComicAtlas Beta v0.1 测试清单

> 目标：验证 MVP 全链路（Import → Task → Library → Comic Detail → Reader → History）是否可用。
> 执行方式：按模块逐项手动验证，发现问题记录在 `docs/issues/`。

---

## ① 导入（Import）

- [ ] ZIP 正常导入：选择 ZIP 类型，输入合法路径，点击"开始导入"后跳转到 `/tasks`
- [ ] DIRECTORY 正常导入：选择目录类型，输入合法目录路径，任务创建成功
- [ ] 路径不存在：任务创建或 Worker 处理阶段报错，前端显示清晰错误信息
- [ ] 重复导入：同一 ZIP/目录再次导入时后端去重逻辑生效，前端收到明确提示
- [ ] 非法路径：空路径或异常字符路径被拒绝，前端显示校验提示
- [ ] 导入失败后 Retry：FAILED 任务点击"重试"后回到 PENDING 并重新进入队列
- [ ] Cancel 生效：PENDING/DOWNLOADING 任务可取消，状态变为 CANCELLED

### 检查点

- 创建任务后 `/tasks` 页面能否立即看到新任务？
- 任务状态轮询是否按 IMPORTING 1s / PENDING 2s 自适应？
- 成功后轮询是否自动停止？
- 失败后轮询是否停止并保留错误信息？

## ⑦ 全链路一次成功

按顺序执行以下完整流程，全部通过才算 Beta v0.1 可用：

```text
导入一本新漫画（ZIP 或 DIRECTORY）
    ↓
Task Center 看到任务 → 状态变为 SUCCESS
    ↓
Library 出现新漫画
    ↓
点击 Card 进入 Comic Detail
    ↓
点击"开始阅读"进入 Reader，翻到第 5 页
    ↓
退出 Reader（返回 History 或 Library）
    ↓
History 中出现该记录，显示第 5 页
    ↓
点击"继续阅读"
    ↓
Reader 回到第 5 页
```

- [ ] 全链路一次通过

## 修复摘要

- **BUG-006**: Worker `Jackson2JsonMessageConverter` 注册 `JavaTimeModule`，解决 `Instant` 反序列化失败。
- **BUG-007**: Worker 增加宿主机路径到容器路径映射；`api-service` 挂载 `metadata`/`hq` 卷以读取 Worker 产出的 `metadata.json`。
- **前端**: `api.ts` 增加响应拦截器，统一解包 `{code, message, data}`；`ReaderLayout` 改为 `<router-view>` 修复阅读器白屏；`ReaderPage` 移除滚动驱动页码、改用键盘翻页 + 300ms 防抖保存进度。
- **E2E**: 新增 `smoke` / `import` / `reader` / `history` 四条用例，全部通过。
